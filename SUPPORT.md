# Support

Use the right channel so bug reports, support requests, and design conversations stay manageable.

## Where to ask what

- Bugs:
  - Open a GitHub issue with reproduction steps, expected behavior, and environment details.
- Feature ideas:
  - Open a feature request issue or a discussion once GitHub Discussions is enabled for the repository.
- Connector proposals:
  - Use the connector submission issue template.
- Usage questions:
  - Prefer GitHub Discussions at launch. Until that is enabled, use issues only if you have confirmed the behavior is a product bug.

## What to include

- ACSA version
- OS and architecture
- workflow snippet or sample input
- logs or error output
- whether the issue reproduces in the CLI, UI, or both

## Response expectations

- Security-sensitive reports should not be filed publicly
- Early release candidates may change quickly; verify on the latest build before reporting
- Maintainers may ask for a minimal reproduction before triage
