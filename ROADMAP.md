# Roadmap

This roadmap is intentionally practical. It should evolve with release feedback, but it should not become a backlog dump.

## Current focus

- Finish release collateral and launch the first public version
- Close remaining plugin-runtime security follow-ups
- Keep YAML and UI authoring flows stable

## Near term

- Phase 10 security hardening and dependency remediation
- stronger subprocess isolation controls
- clearer connector packaging patterns
- broader workflow examples and tutorials

## Medium term

- richer connector ecosystem
- better workflow diffing and YAML round-trip ergonomics
- stronger import or export flows for workflow sharing
- improved AI node provider abstractions and testing harnesses

## Long term

- larger library of production-ready connectors
- stronger team and multi-user operational workflows
- more advanced editor ergonomics while keeping YAML as the source of truth

## Non-goals for the first public release

- managed cloud control plane
- no-code-only workflow authoring
- proprietary connector lock-in
